# 短链接项目：智能投放与分析 Agent 技术清单与配置说明（讨论稿）

> 文档性质：技术选型、组件边界、配置规划  
> 当前任务边界：配置规划不代表当前代码已经具备；后续实现前需再次确认技术栈  
> 核心原则：Agent 能力独立，短链业务事实仍由现有 Java 服务负责  

---

## 1. 技术路线推荐

### 1.1 推荐形态

```text
agent-service：独立服务，负责 Agent Runtime。
admin-agent-adapter：现有 admin 模块新增包，负责工具安全执行。
project：保持现有短链业务与统计事实源。
gateway/nginx：只补必要路由，不承载 Agent 逻辑。
```

### 1.2 技术栈建议

| 组件 | 推荐技术 | 说明 |
|---|---|---|
| Agent Service | Python 3.12 + FastAPI + Pydantic | 适合快速构建工具调用、结构化输出和测试 |
| Admin Adapter | Java 17 + Spring Boot 3 | 复用现有 admin 用户和 Feign |
| 数据库 | 现有 MySQL link 库 | 第一阶段新增 Agent 审计表，暂不分库 |
| 缓存 | Redis | 只用于短期 session cache 或限流，非事实源 |
| LLM | OpenAI-compatible endpoint / 可插拔 | 文档不绑定供应商 |
| Trace | MySQL + 应用日志 | 第一阶段不强制引入 Langfuse |
| 前端 | 现有 dist-link 后续补页面或新前端源码 | 第一阶段可先 API 跑通 |

> 待确认事项：如果你倾向全 Java 实现，Agent Service 可改为新的 Maven module，但需要补 Tool Calling、Prompt、结构化输出和评测设施。

## 4. Agent Service 配置

### 4.1 `.env.example`

```env
# =========================================================
# Agent Service
# =========================================================
AGENT_SERVICE_HOST=0.0.0.0
AGENT_SERVICE_PORT=8010
AGENT_ENV=local

# =========================================================
# ShortLink Admin Adapter
# =========================================================
SHORTLINK_ADMIN_BASE_URL=http://127.0.0.1:8002
SHORTLINK_AGENT_INTERNAL_TOKEN=change-me
SHORTLINK_TOOL_TIMEOUT_MS=5000
SHORTLINK_MAX_BATCH_CREATE_SIZE=20

# =========================================================
# LLM
# =========================================================
LLM_PROVIDER=openai-compatible
LLM_BASE_URL=
LLM_API_KEY=
LLM_MODEL=
LLM_TIMEOUT_MS=30000
LLM_MAX_OUTPUT_TOKENS=2000

# =========================================================
# Trace
# =========================================================
AGENT_TRACE_ENABLED=true
AGENT_LOG_LEVEL=INFO
AGENT_REDACT_SENSITIVE=true
```

### 4.2 配置规则

```text
LLM_API_KEY 不得提交仓库；
internal token 不得写死在代码；
本地、测试、生产必须使用不同配置；
模型名称、prompt profile、tool schema 必须带版本；
feature flag 默认关闭写操作。
```

## 5. Admin 配置

建议：

```yaml
short-link:
  agent:
    enabled: false
    write-enabled: false
    service-url: http://127.0.0.1:8010
    internal-token: change-me
    tool-timeout-ms: 5000
    max-batch-create-size: 20
    access-record-max-page-size: 100
    trace-enabled: true
    risk-scan-enabled: false
```

说明：

| 配置 | 说明 |
|---|---|
| `enabled` | 是否启用 Agent API |
| `write-enabled` | 是否允许确认后写操作 |
| `service-url` | Agent Service 地址 |
| `internal-token` | 内部调用鉴权 |
| `max-batch-create-size` | 批量创建上限 |
| `access-record-max-page-size` | 访问明细最大分页 |
| `risk-scan-enabled` | 风控 Agent 预留 |

## 6. Tool Schema 规范

### 6.1 通用请求头

内部工具调用必须带：

```text
X-Agent-Internal-Token；
X-Agent-Trace-Id；
X-Agent-Session-Id。
```

### 6.2 通用请求体

```json
{
  "username": "current-user-from-admin",
  "traceId": "trace_xxx",
  "payload": {}
}
```

### 6.3 通用响应体

```json
{
  "success": true,
  "data": {},
  "errorCode": null,
  "errorMessage": null,
  "snapshotId": "snap_xxx"
}
```

### 6.4 Tool 结果脱敏

| 字段 | 脱敏规则 |
|---|---|
| IP | `192.168.1.*` |
| UV | 哈希后保留前 8 位 |
| User-Agent | 可只保留浏览器/系统/设备 |
| token | 永不返回 |
| 手机号 | `138****1234` |
| 邮箱 | `a***@domain.com` |

## 7. Prompt Profile 规范

建议文件：

```text
agent-service/app/prompts/campaign_analyst_v1.md
agent-service/app/prompts/safety_policy_v1.md
agent-service/app/prompts/tool_use_policy_v1.md
```

Prompt 必须包含：

```text
Agent 角色；
允许任务；
禁止任务；
工具使用规则；
数据真实性要求；
写操作确认要求；
隐私脱敏要求；
无数据时回答规范；
异常和失败回答规范。
```

禁止：

```text
Prompt 中出现真实 API Key；
Prompt 要求模型绕过工具；
Prompt 要求模型直接执行删除；
把网页标题或用户 describe 当作系统指令。
```

## 8. 日志与 Trace 字段

每次 Agent 运行必须至少记录：

```text
trace_id；
session_id；
username；
intent；
tool_name；
tool_status；
tool_latency_ms；
model_name；
prompt_profile_version；
answer_snapshot_ids；
pending_action_ids；
error_code；
created_at。
```

日志脱敏规则：

```text
不得记录 token；
不得记录完整 LLM_API_KEY；
不得记录完整 IP；
不得记录完整 UV；
不得记录未脱敏访问明细。
```

## 9. 与现有配置的冲突提醒

当前项目存在以下配置风险，后续开发前应处理或至少记录：

| 风险 | 文件 | 建议 |
|---|---|---|
| Redis/Nacos/MySQL 硬编码 | 各 `application.yaml` | 后续迁移为环境变量 |
| Gateway route 缺失 | `gateway/src/main/resources/application.yaml` | 补本地路由或明确 Nacos 配置 |
| Java 编译版本不一致 | `project/pom.xml`、`gateway/pom.xml` | 后续统一到 17 |
| EasyExcel 版本不一致 | 父子 POM | 后续统一版本管理 |
| 高德 Key 明文 | `project/application.yaml` | 后续外置 |

第一阶段 Agent 不强制修复全部问题，但新增配置不得继续扩大硬编码问题。

## 10. 推荐启动顺序

本地开发建议：

```text
1. 启动 MySQL / Redis / Nacos；
2. 启动 project；
3. 启动 admin；
4. 启动 agent-service；
5. 启动 gateway；
6. 启动 nginx 或直接调 admin API；
7. 运行 smoke test。
```

如果使用 `aggregation`：

```text
1. 启动 MySQL / Redis；
2. 启动 aggregation；
3. 启动 agent-service；
4. 通过 admin agent API 联调。
```

> 注意：`aggregation` 当前是否能完整注入 Feign 需要开发阶段验证。
