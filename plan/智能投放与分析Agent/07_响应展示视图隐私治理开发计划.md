# 响应展示视图隐私治理 Implementation Plan

> **For agentic workers:** 本计划承接 `06_洞察解释质量增强开发计划.md`。本阶段只治理 Agent 默认分析响应的展示视图，不取消业务系统未来按权限查看完整访问明细的能力。

## Goal

让 Agent Console 默认消费的 `AgentRunResult` 不直接暴露完整 `ip/user`：

```text
LLM prompt 已脱敏；
Agent 默认响应中的 cards、toolCalls、dataSources[type=tool].executions 也必须脱敏；
access_records.rows/rawData 默认只展示脱敏 IP，并移除 user；
内部工具执行和后续高权限明细能力不在本阶段重构。
```

## Scope

本阶段修改范围：

```text
DefaultCampaignAnalysisGraphExecutor.composeResponse
DefaultCampaignAnalysisGraphExecutorTest
正式版开发文档
正式版验收清单
```

本阶段不做：

```text
不新增业务 API；
不修改 AgentRunResult 字段；
不重构 Tool Facade；
不删除 admin 原有访问明细能力；
不实现高权限 full 明细开关。
```

## Design

Graph 内部仍保存原始工具执行状态，响应出站前生成展示视图：

```text
toolExecutions(raw internal state)
  -> buildCards(raw internal state)
  -> sanitizeForResponse(cards)
  -> sanitizeForResponse(toolExecutions)
  -> sanitizeForResponse(dataSources)
  -> AgentRunResult public view
```

`sanitizeForResponse` 第一版复用与 prompt 一致的敏感字段策略：

```text
key == ip: 192.168.1.10 -> 192.168.*.*
key == user: remove
nested List/Map: recursive sanitize
```

## Test Strategy

使用现有 Graph executor 测试升级访问记录场景：

```text
工具返回 access records: records[0].ip = 127.0.0.1, user = visitor-001
断言 access_records card.rows[0].ip = 127.0.*.*
断言 access_records card.rows[0] 不包含 user
断言 access_records card.rawData.records[0] 同样脱敏
断言 result.toolCalls / result.dataSources / result.toString 不包含 127.0.0.1 或 visitor-001
```

## Acceptance Criteria

- [ ] `cards` 默认展示视图不包含完整 `ip/user`。
- [ ] `toolCalls` 默认展示视图不包含完整 `ip/user`。
- [ ] `dataSources[type=tool].executions` 默认展示视图不包含完整 `ip/user`。
- [ ] `access_records.rows/rawData` 默认只展示脱敏 IP，并移除 user。
- [ ] 不修改 `AgentRunResult` 字段。
- [ ] 不新增业务 API 调用。
- [ ] `mvn -pl agent-service test` 通过。
- [ ] `mvn -pl agent-service -DskipTests package` 通过。
- [ ] 提交前 secret/config/target 检查无命中。
