# 洞察解释质量增强 Implementation Plan

> **For agentic workers:** 本计划承接 `05_异常洞察卡片开发计划.md`。实现时必须保持派生 cards 的确定性，LLM 只做解释，不回写、不覆盖、不生成指标。

## Goal

在 `agent-service` 中为 `traffic_anomaly` 与 `performance_insight` 增加稳定的 LLM 解释契约：

```text
有派生洞察卡片时，Graph prompt 必须要求模型围绕 Derived insight context 输出可能原因、风险级别、证据引用和建议动作；
没有派生洞察卡片时，不注入该解释契约，避免普通查询被过度约束；
模型不得重新计算、编造或覆盖 cards 中的 metrics、thresholds、evidence、type、sourceTool、reasonCode；
异常解释必须保留“不构成确定性安全结论”的边界。
```

## Architecture

本阶段不新增业务 API，不修改 `AgentRunResult`，不修改 Tool Facade。

变更只发生在 Graph LLM prompt 组装层：

```text
toolExecutions
  -> sanitizeForPrompt(toolExecutions)
  -> build deterministic derived insight cards
  -> sanitizeForPrompt(derivedInsightCards)
  -> append Insight explanation contract when derived cards are non-empty
  -> DeepSeek chat
```

## Prompt Contract

`Insight explanation contract` 第一版包含以下固定要求：

```text
Use Derived insight context as the factual source for anomaly and performance explanations.
Do not recalculate, invent, or overwrite card metrics, thresholds, evidence, type, sourceTool, or reasonCode.
For each derived insight, explain possibleCauses, riskLevel, evidenceReferences, and recommendedActions.
Keep riskLevel conservative: high only for multiple strong warning signals; otherwise medium or low.
recommendedActions must be read-only or low-risk operational suggestions.
State that this is not a definitive security conclusion when discussing traffic anomaly cards.
Respond in the user's language unless the user explicitly asks otherwise.
```

## Test Strategy

使用 TDD，先补失败测试：

```text
executeAddsInsightExplanationContractWhenDerivedCardsExist
  - 构造会生成 traffic_anomaly 的 stats 工具结果；
  - 断言 prompt 包含 Insight explanation contract；
  - 断言 prompt 包含 possibleCauses / riskLevel / evidenceReferences / recommendedActions；
  - 断言 prompt 不包含原始完整 IP。

executeSkipsInsightExplanationContractWhenNoDerivedCardsExist
  - 构造不会生成派生 cards 的 stats 工具结果；
  - 断言 prompt 有 Tool execution context；
  - 断言 prompt 不包含 Derived insight context 和 Insight explanation contract。
```

## Acceptance Criteria

- [ ] 有派生洞察卡片时，LLM prompt 注入 `Derived insight context` 与 `Insight explanation contract`。
- [ ] 无派生洞察卡片时，不注入解释契约。
- [ ] 派生洞察上下文进入 prompt 前再次经过 `sanitizeForPrompt`。
- [ ] 契约要求模型不得覆盖 deterministic cards 指标。
- [ ] 契约要求输出可能原因、风险级别、证据引用、建议动作。
- [ ] `traffic_anomaly` 解释必须提示“不是确定性安全结论”。
- [ ] 不新增远程业务调用。
- [ ] 不修改 `AgentRunResult`。
- [ ] `mvn -pl agent-service test` 通过。
- [ ] `mvn -pl agent-service -DskipTests package` 通过。
- [ ] 提交前 secret/config/target 检查无命中。
