# E2E 联调闭环与工具结果治理实施计划

> **For agentic workers:** 本计划承接 `10_admin-gateway正式入口接入计划.md` 和 `11_admin-internal-tool-api接入计划.md`。本阶段目标是把已经完成的正式入口、Graph、Tool Facade、admin internal tool API、Graph checkpoint 串成可验收闭环；自动化测试只使用 mock admin internal API 与 H2 checkpoint，真实 DeepSeek/MySQL 联调通过本地手动脚本验收。

## Goal

建立智能投放与分析 Agent 第一条可重复验证的端到端闭环：

```text
admin /api/short-link/admin/v1/agent/chat
  -> agent-service /internal/short-link-agent/v1/chat
  -> Spring AI Alibaba Graph
  -> agent-service Java Tool Facade
  -> admin /internal/short-link-admin/v1/agent-tools/**
  -> toolCalls / dataSources / cards / traceEvents
  -> JDBC Graph checkpoint
```

本阶段不追求一次性启动全部微服务作为默认测试，而是建立两层验收：

```text
自动化 E2E-ish 回归：MockMvc + real Graph + real Tool + mock admin internal API + H2 checkpoint
真实本机联调：admin、agent-service、MySQL、DeepSeek、短链接业务服务按本地配置启动后用 curl/页面验证
```

测试请求：

```http
POST /internal/short-link-agent/v1/chat
X-Agent-Internal-Token: internal-token
X-Agent-Username: trusted-user
Content-Type: application/json

{
  "sessionId": "e2e-session-1",
  "username": "spoofed-user",
  "message": "show groups and link list and stats and access records gid=g1 startDate=2026-07-01 endDate=2026-07-07 current=1 size=10"
}
```

mock admin internal API 必须验证：

```text
所有工具请求携带 X-Agent-Internal-Token；
所有工具请求携带 X-Agent-Username=trusted-user；
没有普通 username header；
工具路径使用 /internal/short-link-admin/v1/agent-tools/**；
page/stats/access-records query 参数顺序与 Tool Facade 当前实现一致。
```

响应必须验证：

```text
Result.success=true；
data.sessionId=e2e-session-1；
data.answer 来自 LLM stub；
toolCalls 顺序包含 list_groups、page_short_links、get_group_stats、get_group_access_records；
cards 至少包含 group_summary、short_link_page、stats_summary、access_records；
dataSources 包含 graph、llm、tool；
traceEvents 顺序覆盖 intake、tool_planning、llm_analysis、response_compose、checkpoint_save；
checkpoint_save 成功事件包含 checkpointVersion；
访问明细中的 ip/user 在响应、toolCalls、dataSources 中不出现原始明文；
JDBC checkpoint 表写入 FINISHED，checkpoint_json 包含 toolExecutions 和 traceEvents。
```

## 真实本机联调步骤

### 1. 准备未提交配置

环境变量建议：

```powershell
$env:DEEPSEEK_API_KEY = "<local-secret>"
$env:AGENT_INTERNAL_TOKEN = "<local-internal-token>"
$env:SHORT_LINK_BUSINESS_BASE_URL = "http://127.0.0.1:8002"
$env:AGENT_DATASOURCE_URL = "jdbc:mysql://127.0.0.1:3306/shortlink_agent?useUnicode=true&characterEncoding=utf8&serverTimezone=Asia/Shanghai"
$env:AGENT_DATASOURCE_USERNAME = "root"
$env:AGENT_DATASOURCE_PASSWORD = "<local-password>"
```

`agent-service` 本地未提交配置需具备：

```yaml
short-link:
  agent:
    deepseek:
      api-key: ${DEEPSEEK_API_KEY:}
      model: deepseek-v4-flash
    security:
      internal-token: ${AGENT_INTERNAL_TOKEN:}
    business:
      base-url: ${SHORT_LINK_BUSINESS_BASE_URL:http://127.0.0.1:8002}
      internal-token: ${AGENT_INTERNAL_TOKEN:}
    graph:
      checkpoint-enabled: true

spring:
  datasource:
    url: ${AGENT_DATASOURCE_URL}
    username: ${AGENT_DATASOURCE_USERNAME}
    password: ${AGENT_DATASOURCE_PASSWORD}
```

`admin` 本地未提交配置需具备：

```yaml
short-link:
  agent:
    admin:
      remote-url: http://127.0.0.1:8010
      internal-token: ${AGENT_INTERNAL_TOKEN:}
      internal-token-dev-mode: false
```

### 2. 初始化 checkpoint 表

在 agent-service 数据库执行：

```sql
CREATE TABLE IF NOT EXISTS t_agent_graph_checkpoint (
    id BIGINT NOT NULL AUTO_INCREMENT,
    thread_id VARCHAR(128) NOT NULL,
    trace_id VARCHAR(128) NOT NULL,
    graph_name VARCHAR(128) NOT NULL,
    graph_version VARCHAR(64) NOT NULL,
    checkpoint_json LONGTEXT NOT NULL,
    checkpoint_version BIGINT NOT NULL,
    status VARCHAR(32) NOT NULL,
    create_time TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
    update_time TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
    PRIMARY KEY (id),
    CONSTRAINT uk_agent_graph_checkpoint UNIQUE (thread_id, graph_name, graph_version)
);
```

### 3. 启动顺序

```text
MySQL / Redis / Nacos 等基础依赖；
短链接 project 服务，确保统计查询接口有可读数据；
admin 服务，确保 /internal/short-link-admin/v1/agent-tools/** 可被本机访问；
agent-service 服务，确保 /internal/short-link-agent/v1/health 正常；
gateway 服务，确保 /api/short-link/admin/v1/agent/** 走 TokenValidate。
```

### 4. 验证 agent-service 到 admin internal tool API

```powershell
curl -X POST "http://127.0.0.1:8010/internal/short-link-agent/v1/chat" `
  -H "Content-Type: application/json" `
  -H "X-Agent-Internal-Token: $env:AGENT_INTERNAL_TOKEN" `
  -H "X-Agent-Username: zhangsan" `
  -d "{\"sessionId\":\"manual-e2e-1\",\"username\":\"spoofed\",\"message\":\"show groups and link list and stats and access records gid=<gid> startDate=2026-07-01 endDate=2026-07-07 current=1 size=10\"}"
```

验收点：

```text
返回 success=true；
answer 由 DeepSeek 生成；
toolCalls 非空；
dataSources[type=tool].executions 非空；
traceEvents 含 checkpoint_save success；
响应里不出现原始访问明细 ip/user。
```

### 5. 验证 admin 正式入口

通过浏览器或 curl 请求：

```http
POST /api/short-link/admin/v1/agent/chat
```

验收点：

```text
请求必须经过 gateway/admin 登录态；
admin 忽略 body.username，只使用 UserContext；
agent-service 收到 X-Agent-Username；
agent-service 继续调用 admin internal tool API；
最终响应结构与 agent-service internal chat 一致。
```

### 6. 验证 checkpoint

```sql
select thread_id,
       trace_id,
       graph_name,
       graph_version,
       checkpoint_version,
       status,
       json_extract(checkpoint_json, '$.visitedNodes') as visited_nodes
from t_agent_graph_checkpoint
where thread_id = 'manual-e2e-1';
```

验收点：

```text
status=FINISHED；
graph_name=campaign-analysis-graph；
checkpoint_json 包含 toolExecutions、traceEvents、answer；
checkpoint_json 不包含原始访问明细 ip/user。
```

## Discussion Notes

本阶段暂不把真实 DeepSeek 和真实 MySQL 作为默认自动化测试依赖，因为这会让单元回归受本机密钥、网络、数据库状态和业务数据影响。自动化测试验证的是代码链路和协议契约；真实联调验证的是环境配置、网络连通性、模型返回和真实业务数据质量。两者都需要，但职责不同。

后续如果引入 Testcontainers 或统一本地编排脚本，可以把 MySQL checkpoint 和 admin/project mock 服务提升到更接近完整微服务级别的集成测试；在当前 Windows 本地与传统项目依赖较多的情况下，先保留轻量、稳定、可重复的 E2E-ish 自动化测试。
