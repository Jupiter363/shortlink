## 2. 当前基线

截至 2026-07-10，仓库已具备：

```text
agent-service 顶级 Maven module
Spring AI Alibaba Graph 两条独立 Graph
CampaignAnalysis Agent 只读统计与洞察卡片
SecurityRisk Agent 风险规则、LLM 解释、风险事件和自动 LIMIT_RATE
risk-profile 两小时画像、近 7 天活跃扫描、分组画像和 7 天趋势
risk-center 查询、快照、事件、审核记录
risk-policy MySQL 持久化和 Redis 发布
gateway Redis 热路径拦截
admin Agent 正式入口、风险查询与审核 facade
MySQL/H2 Repository 测试、Graph 测试和模块级测试
```

当前最重要的缺口不是新增更多 Graph 节点，而是将上述组件连接成可重试、可恢复、可审计、可限额的生产运行链路。

## 3. 执行原则

```text
1. 不把画像全量计算交给 LLM。
2. 每个风险批次每个 gid 最多运行一次 SecurityRisk Graph。
3. 不让定时画像线程同步等待所有 LLM 调用。
4. LLM 只生成解释和动作提案，写操作必须经过确定性校验。
5. LIMIT_RATE 仅按既定规则自动执行。
6. DISABLE_SHORT_LINK、BLOCK_IP、LIMIT_TIME_WINDOW 必须人工确认。
7. Campaign 创建短链和批量创建必须先生成待确认动作。
8. Gateway 继续只读 Redis，不调用 MySQL 或 agent-service。
9. 原始 IP、用户、visitor、密钥、数据库地址不得进入 prompt、响应、日志或 checkpoint。
10. 每完成一个可测试阶段，运行测试、提交并推送。
```
