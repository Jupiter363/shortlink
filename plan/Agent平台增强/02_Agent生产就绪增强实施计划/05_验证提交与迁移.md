# 验证、提交与迁移 Implementation Plan

## 0. 当前验证边界

本阶段执行本地自动化验证，不执行真实外部 E2E：

```text
不启动 DeepSeek。
不连接真实 MySQL/Redis。
不启动 admin/project/gateway/agent-service 完整服务链。
不执行 scripts/risk-profile-policy-e2e.ps1 的真实网络闭环。
```

真实 DeepSeek + MySQL checkpoint + admin/project/internal tool API 联调保留为后续独立阶段，不能用本地单元测试结果替代。

## 1. 模块验证

```bash
mvn -q -pl agent-service test
mvn -q -pl admin test
mvn -q -pl gateway test
mvn -q -pl project test
```

关键专项覆盖：

```text
批次 owner lease 并发竞争与过期接管。
画像写入、分组聚合和任务发布的租约 fence。
FAILED/过期 RUNNING 批次恢复与坏候选公平轮转。
RISK_PROFILE_FAILED_RECOVERY_LIMIT 的 Spring Binder 绑定。
Graph NOT_REQUESTED/AVAILABLE/NO_DATA/SOURCE_FAILURE 证据语义。
批处理 LLM fail-fast、交互模式降级和 checkpoint 副作用。
```

## 2. 数据库验证

必须确认：

```text
新表在 MySQL 8 和 H2 MySQL mode 均可创建。
所有新唯一键与 Repository upsert 语义一致。
旧风险画像、事件、策略和 checkpoint 数据仍可读取。
原 t_agent_risk_policy 继续保留历史记录。
原 t_agent_graph_checkpoint 作为脱敏运行快照兼容保留。
Spring AI Alibaba MysqlSaver 使用独立框架表，不覆盖业务快照表。
恢复队列索引与 findRecoverableBefore 的状态、排序和 lease 条件一致。
```

提供增量 SQL：

```text
agent-service/src/main/resources/sql/migration/V20260710__agent_production_readiness.sql
```

恢复索引迁移规则：

```text
agent_service_schema.sql：
  新库在 t_agent_risk_profile_batch 的 CREATE TABLE 中声明
  idx_agent_risk_profile_batch_recovery
  (update_time, window_end, id, status, lease_until)。

V20260710__agent_production_readiness.sql：
  CREATE TABLE IF NOT EXISTS 中不重复声明该新索引；
  表创建后使用一次显式 ALTER TABLE ... ADD INDEX，
  确保存量 t_agent_risk_profile_batch 也能获得索引。
```

迁移执行前必须暂停所有 risk-profile scheduler 和 risk-analysis worker。该项目未接入 Flyway/Liquibase；`t_agent_schema_migration_history` 已存在 `V20260710` 时禁止重跑脚本。

## 5. 最终代码审查

审查顺序：

```text
1. 生产编排是否只消费指定 batch。
2. 所有状态机是否具备幂等和非法状态保护。
3. 所有写动作是否经过 pending action 或确定性自动规则。
4. Redis 撤销是否包含版本保护。
5. prompt/response/checkpoint/run log 是否统一脱敏。
6. session/thread 是否按用户和 agentType 隔离。
7. Tool 参数是否受 schema、分页和日期预算限制。
8. LLM 失败是否不破坏确定性画像、事件和策略逻辑。
9. Tool 成功零记录是否保持 success=true，并进入 NO_DATA 而不是 SOURCE_FAILURE。
10. 恢复失败候选是否更新 attempt 时间，避免长期占据 recovery limit。
```

全部验证通过后提交并推送当前分支。
