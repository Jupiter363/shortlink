# 验证、提交与迁移 Implementation Plan

## 1. 模块验证

```bash
mvn -pl agent-service test
mvn -pl admin test
mvn -pl gateway test
mvn -pl project test
```

## 2. 数据库验证

必须确认：

```text
新表在 MySQL 8 和 H2 MySQL mode 均可创建。
所有新唯一键与 Repository upsert 语义一致。
旧风险画像、事件、策略和 checkpoint 数据仍可读取。
原 t_agent_risk_policy 继续保留历史记录。
原 t_agent_graph_checkpoint 作为脱敏运行快照兼容保留。
Spring AI Alibaba MysqlSaver 使用独立框架表，不覆盖业务快照表。
```

提供增量 SQL：

```text
agent-service/src/main/resources/sql/migration/V20260710__agent_production_readiness.sql
```

## 3. 安全验证

```bash
rg -n "sk-[A-Za-z0-9]{16,}|Bearer\\s+[A-Za-z0-9._-]{16,}|DEEPSEEK_API_KEY\\s*[:=]\\s*sk-|AGENT_INTERNAL_TOKEN\\s*[:=]\\s*[^}\\s]+|RISK_HASH_SALT\\s*[:=]\\s*[^}\\s]+|rawIp|ipAddress|access_records\\.rawData" agent-service admin gateway project plan scripts
```

人工核查允许出现的字段名测试和禁止出现的真实值。

## 4. Git 验证

```bash
git diff --check
git status --short --branch
git diff --stat
```

禁止提交：

```text
application.yaml
application-*.yaml
.env
真实 API key
真实数据库密码
真实 internal token
真实 hash salt
```

## 5. 最终代码审查

审查顺序：

```text
1. 生产编排是否只消费指定 batch。
2. 所有状态机是否具备幂等和非法状态保护。
3. 所有写动作是否经过 pending action 或确定性自动规则。
4. Redis 撤销是否包含版本保护。
5. prompt/response/checkpoint/run log 是否统一脱敏。
6. session/thread 是否按用户和 agentType 隔离。
7. Tool 参数是否受 schema、分页和日期预算限制。
8. LLM 失败是否不破坏确定性画像、事件和策略逻辑。
```

全部验证通过后提交并推送当前分支。
