# 待确认动作与策略一致性 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 建立可持久化、可确认、可拒绝、可并发保护的通用 Agent 动作中心，并让风控策略通过有效槽位和 Redis Outbox 可靠生效。

**Architecture:** Harness 只管理动作生命周期，SecurityRisk Agent 只生成结构化提案，风控执行器通过事务化策略服务写 history、effective slot、review、audit 和 outbox。Admin 在所有权校验后开放确认入口，Gateway 继续只读 Redis。

**Tech Stack:** Java 17、Spring Boot 3.0.7、Spring JDBC、Spring Transaction、Spring AI Alibaba Graph 1.1.2.3、MySQL 8、H2 MySQL Mode、Spring Data Redis、Spring Cloud OpenFeign、JUnit 5、AssertJ、Mockito。

## 3. 依赖顺序

```text
02A 通用动作状态机
  -> 02B SecurityRisk 生成持久化提案
  -> 02C 注册安全执行器并启用策略一致性
  -> 02D Admin 才开放正式确认入口
  -> 02E 全量验证
```

禁止跳过 02C 直接开放 02D。否则确认动作可能落到旧的“MySQL 后同步 Redis”路径。
