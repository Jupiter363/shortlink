# 02A 通用待确认动作核心 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 在 `agent-service` 中建立持久化动作模型、提案幂等、活跃槽位、确认/拒绝状态机、执行 fencing、执行器注册表和可信内部 API。

**Architecture:** `harness/action` 不依赖风控或 Campaign。Repository 使用 H2/MySQL 均可执行的条件更新实现 CAS；Service 在短事务中抢占动作，在事务外调用 executor，再用 execution token 完成终态更新。

**Tech Stack:** Spring JDBC、Jackson、Spring MVC、MySQL 8、H2 MySQL Mode、JUnit 5、AssertJ、Mockito。

```text

@Test
void onlyOneConcurrentConfirmationClaimsTheAction() throws Exception {
    Fixture fixture = fixture("pending_action_claim_concurrency");
    fixture.insertPending("action-1", 1L);
    ExecutorService executor = Executors.newFixedThreadPool(2);
    CountDownLatch ready = new CountDownLatch(2);
    CountDownLatch start = new CountDownLatch(1);

    Callable<Optional<AgentActionClaim>> claim = () -> {
        ready.countDown();
        start.await(5, TimeUnit.SECONDS);
        return fixture.repository.claimForExecution(
                "action-1",
                1L,
                "token-" + Thread.currentThread().getId(),
                NOW,
                Duration.ofMinutes(5),
                true
        );
    };
    Future<Optional<AgentActionClaim>> first = executor.submit(claim);
    Future<Optional<AgentActionClaim>> second = executor.submit(claim);
    assertThat(ready.await(5, TimeUnit.SECONDS)).isTrue();
    start.countDown();

    long winners = Stream.of(first.get(), second.get()).filter(Optional::isPresent).count();
    assertThat(winners).isEqualTo(1L);
    executor.shutdownNow();
}
```

`AgentActionClaim` 必须明确保存抢占后的版本：

- [ ] **Step 5: 实现授权和 confirm/reject 流程**

`AgentActionAuthorizationService`：

```text
GID：actor.expectedGid 必须非空并等于 action.gid。
OWNER：actor.username 必须等于 action.ownerUsername。
任何失败都抛 ACTION_SCOPE_FORBIDDEN。
```

`AgentPendingActionService.confirm()`：

```text
读取动作并授权。
EXECUTED 直接返回 view。
EXECUTING 返回当前 view。
查找 executor；不存在抛 ACTION_EXECUTOR_UNAVAILABLE。
按 executor.replaySafe() 抢占。
调用 executor。
使用 token/version 写 EXECUTED。
异常先脱敏，再按 token/version 写 FAILED，最后抛 ACTION_EXECUTION_FAILED。
```

Service 公开提案入口固定为：

```java
AgentPendingActionView propose(AgentActionProposal proposal, Duration ttl);
AgentPendingActionView detail(String actionId, AgentActionActor actor);
AgentActionPage page(
        String gid,
        String agentType,
        String actionType,
        AgentActionStatus status,
        int pageNo,
        int pageSize,
        AgentActionActor actor
);
AgentPendingActionView confirm(
        String actionId,
        long expectedVersion,
        AgentActionActor actor,
        String note
);
AgentPendingActionView reject(
        String actionId,
        long expectedVersion,
        AgentActionActor actor,
        String reason,
        String reviewAction
);
```

它校验 `ttl` 为正数，通过注入的 `Clock` 计算过期时间，并把 Repository 的 `AgentActionPayloadConflictException` 原样传播。`confirm()` 使用 `AgentProperties.Action.executionLeaseSeconds`，禁止在 Service 内写死租约。

`reject()` 只允许 `PENDING` 或 `FAILED`，只接受空值、`IGNORE` 或 `FALSE_POSITIVE`，拒绝后持久化 `rejectionReviewAction` 并释放 active slot。
