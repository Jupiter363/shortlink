# 安全风控 Agent 正式版开发文档

## 1. 第一阶段目标

在 `agent-service` 中新增 `securityriskagent`，实现一条独立的 SecurityRisk Graph。它与现有 `campaignanalysisagent` 分包隔离，但复用 harness、Tool Registry、Business Gateway、LLM client 和 checkpoint。

## 2. Agent 类型

| agentType | Agent | 说明 |
|---|---|---|
| 空 | `campaignanalysisagent` | 兼容已有调用 |
| `campaign-analysis` | `campaignanalysisagent` | 智能投放与分析 |
| `security-risk` | `securityriskagent` | 安全风控诊断 |

## 3. SecurityRisk Graph 节点

```text
intake：
  初始化 graphName、graphVersion、agentType、sessionId、username。

risk_tool_planning：
  从 message 提取 gid、fullShortUrl、startDate、endDate、current、size。
  根据关键词和参数调用 get_group_stats / get_short_link_stats / get_group_access_records。

risk_scoring：
  使用 SecurityRiskCardFactory 对工具结果执行确定性规则。
  输出 risk_summary、risk_signal、risk_evidence、risk_recommendation cards。

llm_explanation：
  只把脱敏后的 risk cards 和 tool summary 送入 LLM。
  要求 LLM 不重算指标、不夸大攻击结论、不输出原始敏感数据。

response_compose：
  返回 answer、cards、toolCalls、dataSources、traceEvents、warnings、pendingActions。
```

## 5. PendingAction 策略

第一阶段只生成建议，不执行写操作：

| pendingAction | 触发场景 | 执行状态 |
|---|---|---|
| `review_ip_concentration` | IP 集中高风险 | 待用户确认 |
| `review_channel_source` | 来源画像集中 | 待用户确认 |
| `tighten_flow_limit` | 后台操作频率异常 | 待用户确认 |
| `disable_short_link_candidate` | 单短链疑似恶意传播 | 待用户确认 |

后续如果要执行写动作，必须由 admin 新增正式 API，并保留审计。

## 6. 测试策略

```text
DefaultAgentRunHarnessTest：
  验证默认路由 campaign-analysis。
  验证 security-risk 路由到 SecurityRiskGraphExecutor。

AgentChatControllerTest：
  验证 agentType 从请求体进入 AgentRunRequest。

SecurityRiskCardFactoryTest：
  验证 Top IP 集中、PV/UV 过高、小时集中、画像集中。
  验证 IP/user 脱敏。

DefaultSecurityRiskGraphExecutorTest：
  验证工具规划、LLM prompt 脱敏、checkpoint graphName 隔离。
```
