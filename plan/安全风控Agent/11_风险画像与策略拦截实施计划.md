# 风险画像与策略拦截 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 构建风险画像、风险策略、Redis 热路径拦截、SecurityRisk Agent 精准诊断、人工审核与 admin 查询入口的完整后端闭环。

**Architecture:** `agent-service` 负责风险画像、风险事件、风险快照、策略落库、Redis 策略发布和 SecurityRisk Agent 编排；`gateway` 在短链跳转热路径只读 Redis 拦截，不调用 MySQL 和 agent-service；`admin` 只做面向后台页面的风险查询与审核 facade，不把风险查询封装成 Agent Tool。

**Tech Stack:** Java 17、Spring Boot 3.0.7、Spring Cloud Gateway、Spring AI Alibaba Graph、DeepSeek V4 Flash、JdbcTemplate、MySQL/H2、Redis/StringRedisTemplate、JUnit 5、WebTestClient、MockMvc。

---

## 0. 执行边界

本计划承接 `plan/安全风控Agent/10_风险画像与策略拦截增强设计.md`，不重新讨论方案选择。实施时每完成一个可测试阶段就运行本阶段测试、提交、推送。

关键约束：

```text
1. 风险画像计算不调用 LLM。
2. 风险查询 API 不走 Agent，不走 Tool，不走 LLM。
3. SecurityRisk Agent 只处理画像筛出的异常候选和组级解释。
4. Gateway 只读 Redis，不查 MySQL，不调 agent-service。
5. 原始 IP/user/visitor 不进入画像表、风险事件表、风险快照表、LLM prompt、LLM response、checkpoint。
6. DISABLE_SHORT_LINK 不修改 t_link.enable_status，只通过风控策略在 gateway 生效。
7. LIMIT_RATE 满足确定性条件时才允许自动执行；DISABLE_SHORT_LINK、BLOCK_IP、LIMIT_TIME_WINDOW 必须人工确认。
```

package com.nageoffer.shortlink.gateway.config;

import org.junit.jupiter.api.Test;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.boot.test.context.SpringBootTest;
import org.springframework.cloud.gateway.route.RouteDefinition;
import org.springframework.cloud.gateway.route.RouteDefinitionLocator;
import org.springframework.test.context.ActiveProfiles;
import reactor.core.publisher.Flux;

import java.util.List;

import static org.assertj.core.api.Assertions.assertThat;

@SpringBootTest(properties = {
        "spring.cloud.nacos.discovery.enabled=false",
        "spring.autoconfigure.exclude=org.springframework.boot.autoconfigure.data.redis.RedisAutoConfiguration,"
                + "org.springframework.boot.autoconfigure.data.redis.RedisRepositoriesAutoConfiguration"
})
@ActiveProfiles("risk-local")
class GatewayRiskRouteConfigTest {

    @Autowired
    private RouteDefinitionLocator routeDefinitionLocator;

    @Test
    void redirectRouteHasRiskPolicyFilterAndApiRoutesHaveHigherPriority() {
        List<RouteDefinition> routes = Flux.from(routeDefinitionLocator.getRouteDefinitions()).collectList().block();

        RouteDefinition redirect = find(routes, "short-link-redirect");
        RouteDefinition adminApi = find(routes, "short-link-admin-api");
        RouteDefinition projectApi = find(routes, "short-link-project-api");

        assertThat(redirect.getOrder()).isEqualTo(100);
        assertThat(redirect.getFilters()).anyMatch(filter -> filter.getName().equals("RiskPolicy"));
        assertThat(adminApi.getOrder()).isLessThan(redirect.getOrder());
        assertThat(projectApi.getOrder()).isLessThan(redirect.getOrder());
    }

    private RouteDefinition find(List<RouteDefinition> routes, String id) {
        return routes.stream()
                .filter(route -> route.getId().equals(id))
                .findFirst()
                .orElseThrow();
    }
}
