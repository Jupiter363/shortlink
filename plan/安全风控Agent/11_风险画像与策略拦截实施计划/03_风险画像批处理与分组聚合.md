# 风险画像批处理与分组聚合 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (- [ ]) syntax for tracking.

**Goal:** 交付短链画像、风险评分、近 7 天活跃短链扫描、每 2 小时批处理、分组画像、Top 10 异常候选和 7 天趋势。

**Architecture:** 本批在 agent-service 内建立 riskprofile 子域。画像计算只使用规则和统计，输出脱敏事实，供页面查询和 SecurityRisk Agent 精准消费。

**Tech Stack:** Java 17、Spring Boot 3.0.7、JUnit 5、MySQL/H2、Redis；按本批涉及模块额外使用 Spring Cloud Gateway、Spring AI Alibaba Graph、MockMvc 或 WebTestClient。


```text

```

- [ ] **Step 5: 实现 RiskProfileBatchService**

职责：

```text
1. since = batchNow - activeScanDays。
2. listActiveShortLinks(since)。
3. 按 gid 分组。
4. 为每条短链生成画像。
5. 每组筛选 riskScore desc 的 Top 10 异常短链。
6. 返回 RiskProfileBatchResult。
```

- [ ] **Step 6: 实现 RiskProfileScheduler**

第一版使用 `@Scheduled(fixedDelayString = "${short-link.agent.risk.profile.batch-interval-minutes:120}000")` 不正确，因为分钟无法直接拼毫秒。使用服务方法计算毫秒：

```java
@Scheduled(fixedDelayString = "#{@riskProfileScheduleDelayMillis}")
public void runRiskProfileBatch() {
    batchService.runOnce(clock.instant());
}
```

或者使用固定 cron：

```java
@Scheduled(cron = "0 0 */2 * * ?")
```

本项目第一版采用 cron，每 2 小时整点执行。

- [ ] **Step 3: 实现聚合规则**

规则：

```text
groupRiskScore = max(maxRiskScore, round(avgRiskScore + highRiskCount * 5 + mediumRiskCount * 2))，上限 100。
groupRiskLevel = RiskLevel.fromScore(groupRiskScore)。
groupReasonCodes = 组内 reasonCodes 出现频次倒序，最多 5 个。
topRiskShortLinks = riskScore desc, pv2h desc, shortUri asc，最多 10 个。
riskTrend7d = repository.findTrend7d(gid, profileWindowEnd)，不足 7 天时用已有历史点，不编造空点。
```
