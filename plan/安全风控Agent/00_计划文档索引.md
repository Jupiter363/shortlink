## 已确认结论

```text
方案：B，在 agent-service 内新增第二条 SecurityRisk Graph
包结构：按业务 Agent 隔离
  campaignanalysisagent：智能投放与分析 Agent
  securityriskagent：安全风控 Agent
入口：复用 /internal/short-link-agent/v1/chat，通过 agentType 路由
默认行为：agentType 为空时仍走 campaign-analysis
风控行为：agentType=security-risk 时走 security-risk
工具边界：第一阶段复用现有只读 Tool Facade
处置策略：LIMIT_RATE 可在高风险且证据充分时自动执行；DISABLE_SHORT_LINK、BLOCK_IP、LIMIT_TIME_WINDOW 必须人工确认
敏感数据：IP、user、访问明细继续脱敏后进入 LLM、响应和 checkpoint
风险画像：risk-profile 负责短链画像和分组画像，纯规则/统计，可测试、可重跑
策略拦截：agent-service 管策略，Redis 发布生效策略，gateway 在短链跳转热路径只读 Redis 拦截
```

## 第一阶段交付顺序

```text
1. 固化文档与包结构命名。
2. 增加 agentType 路由，默认兼容旧调用。
3. 将现有 Campaign Analysis Graph 迁移到 campaignanalysisagent 包。
4. 新增 securityriskagent Graph、风险卡片工厂和单元测试。
5. 复用现有 get_group_stats / get_short_link_stats / get_group_access_records 工具。
6. 运行 agent-service 单元测试和敏感信息扫描。
```

## 第二阶段交付顺序

```text
1. 抽取 securityriskagent/safety，统一 LLM、响应和 checkpoint 脱敏出口。
2. 抽取 securityriskagent/model 与 rule，让确定性规则独立可测。
3. 抽取 securityriskagent/prompt，固定安全 prompt 合同。
4. 抽取 securityriskagent/node，让 GraphExecutor 聚焦编排、trace 和 checkpoint。
5. 保持 agentType=security-risk 对外响应结构兼容。
6. 完成 agent-service 全量测试、敏感信息扫描、提交并推送。
```

## 第三阶段交付顺序

```text
1. 在 Agent Console 增加 agentType 选择器。
2. 保持默认 campaign-analysis，避免影响既有本地联调。
3. 支持选择 security-risk 并随 chat 请求提交给 agent-service。
4. 复用现有 cards、pendingActions、traceEvents、Sanitized data 展示。
5. 用静态资源测试锁定 Console 可见控件和请求体字段。
```

## 第四阶段交付顺序

```text
1. 通过 /internal/short-link-agent/v1/chat 发起 agentType=security-risk 请求。
2. 经 InternalAgentApiFilter 验证 internal token 与可信用户头。
3. 经 AgentRunHarness 路由到 SecurityRisk Graph，不误走 campaign-analysis。
4. 调用 admin internal tool API 的 get_group_stats / get_group_access_records。
5. 使用 LLM stub 验证 prompt 已脱敏并包含 Risk signal context。
6. 保存 MySQL checkpoint，graphName=security-risk-graph，checkpoint 和 trace 均可追溯。
```

## 第五阶段交付顺序

```text
1. 补齐 gateway dev/local 可验证路由基线，让短链跳转 /{shortUri} 可挂 RiskPolicy 过滤器。
2. 新增 RiskPolicyGatewayFilterFactory，基于 Redis 策略拦截 DISABLE_SHORT_LINK、BLOCK_IP、LIMIT_RATE、LIMIT_TIME_WINDOW。
3. 在 agent-service 新增 risk-policy、risk-center、risk-profile 相关表结构和服务边界。
4. 实现每 2 小时扫描近 7 天有访问短链的风险画像批处理。
5. 聚合同批短链画像生成分组画像、7 天风险趋势和 Top 10 异常短链。
6. 让 SecurityRisk Agent 消费画像，只处理异常候选并写风险事件、快照和处置建议。
7. 接入人工审核，确认后发布策略到 Redis；LIMIT_RATE 满足确定性条件时允许 Agent 自动执行。
8. 完成 gateway、agent-service、admin 接入测试和敏感信息扫描。
```
