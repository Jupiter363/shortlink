## 已确认结论

```text
方案：B，在 agent-service 内新增第二条 SecurityRisk Graph
包结构：按业务 Agent 隔离
  campaignanalysisagent：智能投放与分析 Agent
  securityriskagent：安全风控 Agent
入口：复用 /internal/short-link-agent/v1/chat，通过 agentType 路由
默认行为：agentType 为空时仍走 campaign-analysis
风控行为：agentType=security-risk 时走 security-risk
工具边界：第一阶段复用现有只读 Tool Facade
处置策略：只生成 pendingActions，不自动执行高风险写操作
敏感数据：IP、user、访问明细继续脱敏后进入 LLM、响应和 checkpoint
```
