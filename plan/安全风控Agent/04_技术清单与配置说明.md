# 安全风控 Agent 技术清单与配置说明

## 1. 技术栈

```text
Java 17
Spring Boot
Spring AI Alibaba Graph
DeepSeek V4 Flash
MySQL checkpoint
JUnit 5
AssertJ
```

## 2. 配置项

沿用现有配置：

```yaml
short-link:
  agent:
    graph:
      checkpoint-enabled: true
    business:
      base-url: ${SHORT_LINK_BUSINESS_BASE_URL:http://127.0.0.1:8002}
      internal-token: ${AGENT_INTERNAL_TOKEN:}
    deepseek:
      api-key: ${DEEPSEEK_API_KEY:${LLM_API_KEY:}}
      base-url: ${LLM_BASE_URL:https://api.deepseek.com}
      model: ${LLM_MODEL:deepseek-v4-flash}
```

新增 Graph 名称常量不需要配置：

```text
campaign-analysis-graph
security-risk-graph
```

## 3. API 请求示例

```http
POST /internal/short-link-agent/v1/chat
X-Agent-Internal-Token: <local-token>
X-Agent-Username: zhangsan
Content-Type: application/json
```

```json
{
  "sessionId": "security-risk-local-1",
  "agentType": "security-risk",

```

## 4. 本地资源策略

当前机器资源紧张时，第一阶段只运行单元测试，不要求启动完整 E2E：

```text
mvn -pl agent-service test
```

真实联调需要：

```text
MySQL 3306
Redis 6379
project 8001
admin 8002
agent-service 8010
DeepSeek API Key
```

## 5. 敏感数据策略

```text
IP：响应、LLM prompt、checkpoint 中只允许脱敏展示。
user：响应、LLM prompt、checkpoint 中不展示原值。
internal token：不进入日志、响应、checkpoint。
DeepSeek API Key：只从环境变量读取，不写入代码和文档。
```
