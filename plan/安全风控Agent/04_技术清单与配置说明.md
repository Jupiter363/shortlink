## 3. Graph 证据语义

SecurityRisk Graph 统一使用四态证据分类：

| 状态 | 含义 | LLM 行为 |
| --- | --- | --- |
| `NOT_REQUESTED` | 当前对话没有请求业务证据 | 普通对话可调用 LLM |
| `AVAILABLE` | Tool 或画像返回可用数据 | 基于脱敏证据调用 LLM |
| `NO_DATA` | Tool 成功但返回空对象、空集合、空字符串或零长度数组 | 不调用 LLM，返回无证据提示 |
| `SOURCE_FAILURE` | 请求的数据源真实调用失败 | 不调用 LLM，返回数据源不可用降级提示 |

Tool 返回零记录时必须保留 `success=true`。只有网络、协议、权限或 Tool 执行异常才进入 `SOURCE_FAILURE`。

规划节点写入的 `evidenceStatus` 只是阶段性派生值。风险评分生成 `riskCards` 后，LLM 解释节点必须基于最终 `toolExecutions + riskCards + evidenceRequested` 重新分类，避免旧 `NO_DATA/SOURCE_FAILURE` 覆盖后续已经产生的有效证据。

## 4. 数据库对象与迁移

业务表与 Spring AI Alibaba Graph checkpoint 表分开管理：

```text
t_agent_risk_profile_batch       风险画像批次与 owner lease
t_agent_risk_analysis_job        异常分组 Agent 分析任务
t_agent_short_link_risk_profile  单短链规则画像
t_agent_group_risk_profile       分组规则画像与 7 天趋势
t_agent_graph_checkpoint         应用侧脱敏 Graph 运行快照
Spring AI Alibaba MysqlSaver 表   框架 checkpoint，不覆盖业务表
```

恢复查询索引：

```sql
idx_agent_risk_profile_batch_recovery
    (update_time, window_end, id, status, lease_until)
```

存量 MySQL 8 使用：

```text
agent-service/src/main/resources/sql/migration/V20260710__agent_production_readiness.sql
```

迁移脚本必须在 scheduler/worker 全部暂停后执行一次，不得在已有 `V20260710` 记录时重跑。

## 6. 本地资源策略

当前机器资源紧张时，第一阶段只运行单元测试，不要求启动完整 E2E：

```text
mvn -q -pl agent-service test
mvn -q -pl admin test
mvn -q -pl gateway test
mvn -q -pl project test
```

真实联调需要：

```text
MySQL 3306
Redis 6379
project 8001
admin 8002
agent-service 8010
DeepSeek API Key
```

当前生产就绪收尾阶段明确不执行上述真实联调，真实 DeepSeek + MySQL checkpoint + internal tool API 闭环留到独立 E2E 阶段。

## 7. 敏感数据策略

```text
IP：响应、LLM prompt、checkpoint 中只允许脱敏展示。
user：响应、LLM prompt、checkpoint 中不展示原值。
internal token：不进入日志、响应、checkpoint。
DeepSeek API Key：只从环境变量读取，不写入代码和文档。
```
